# FastAPI Template - Project Guidelines

This file provides guidance to AI coding agents (Claude Code, Codex, Cursor, etc.) when working with code in this repository.

## Overview

FastAPI + Python 3.14 + PostgreSQL + SQLAlchemy async + Alembic. Hexagonal DDD architecture with bounded contexts under `src/contexts/`. Dependency injection via `dependency-injector`. CLI via Typer.

## Commands

```bash
make start               # Start services (docker compose up)
make stop                # Stop services
make build               # Build Docker images
make fmt                 # Format code with ruff
make lint                # Lint code with ruff --fix
make all                 # Install + format + lint
make test                # Run all tests (Docker)
make test-unit           # Run unit tests (Docker)
make test-unit-local     # Run unit tests locally (fast, no Docker)
make test-integration    # Run integration tests (Docker)
make test-e2e            # Run e2e tests (Docker)
make test-cov            # Run tests with coverage (Docker)
make migration-create    # Create autogenerated Alembic migration
make migration-upgrade   # Apply migrations (upgrade head)
make migration-downgrade # Revert last migration
make migration-history   # Show migration history
make cli args="..."      # Run CLI commands in container
```

## Code Style

- **Language:** All code, variable names, class names, function names, and comments in English. Commit messages and PR descriptions in English.
- **Comments:** Never add comments unless explicitly requested or the code is genuinely non-obvious. Do not add explanatory comments when removing dead code — just remove it.
- **Imports:** Absolute, starting with `src/` (e.g., `from src.contexts.auth.domain.aggregates import User`)
- **Async:** Always `async/await` for I/O operations. Use `asyncpg` driver.
- **Python:** Strict typing. Use `|` union syntax, not `Optional`. Explicit return types on all functions. No `Any`.
- **Errors:** Throw custom Domain Errors (`src/contexts/*/domain/errors.py`), never generic HTTPException in domain/application layers. The HTTP middleware translates domain errors to HTTP responses.
- **Formatting:** Ruff enforces 88-char line length, double quotes, 4-space indent. Do NOT fight ruff — if it reformats something, that's the project standard.

## Git

## Worktrees

Use Claude Code's native worktrees: `claude --worktree <name>` (checkouts live in `.claude/worktrees/`). `.worktreeinclude` copies the gitignored `secrets/` folder into each new worktree automatically.

## README

Keep `README.md` up to date whenever changes affect:

- Available `make` targets (added, removed, or renamed)
- Environment variable setup or the `secrets/` workflow
- Test commands or prerequisites
- New developer tooling (hooks, worktrees, etc.)
- Deployment or CI/CD changes

Do not update README for internal refactors, bug fixes, or changes that don't affect how a developer sets up or uses the project.

## Architecture

Three layers per bounded context: **Domain** (pure Python, no infra deps), **Application** (use cases, inject ports), **Infrastructure** (SQLAlchemy repos, HTTP middleware, CLI, DI containers). See `.claude/rules/architecture.md` for patterns.

## Domain Services

Domain services must always be classes, never standalone functions. Use `@staticmethod` for stateless operations.
